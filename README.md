# Социальная сеть - Лабораторная работа

## Описание проекта

Данная лабораторная работа представляет собой разработку программного обеспечения для социальной сети с использованием ASP.NET Core Minimal API и PostgreSQL базы данных.

## Предметная область

Система представляет собой социальную сеть, где зарегистрированные пользователи могут:
- Публиковать посты и отвечать на посты других пользователей
- Ставить лайки и добавлять посты в избранное
- Подписываться на других пользователей
- Общаться в личных чатах
- Пожаловаться на пользователей или посты
- Администраторы могут модерировать контент и выдавать баны

## Архитектура системы

### Технологии
- **Backend**: ASP.NET Core 9.0 Minimal API
- **База данных**: PostgreSQL
- **ORM**: Dapper (SQL-first подход)
- **Аутентификация**: JWT токены
- **Документация API**: Swagger/OpenAPI

### Структура проекта
```
SocNet.Api/
├── Api/                    # API контроллеры
│   ├── AuthApi.cs         # Аутентификация и регистрация
│   ├── UserApi.cs         # Управление пользователями
│   ├── PostApi.cs         # Посты, лайки, избранное
│   ├── SubscriptionApi.cs # Подписки
│   ├── ChatApi.cs         # Чаты и сообщения
│   └── ReportApi.cs       # Жалобы и модерация
├── Entities/              # Модели данных
├── SQL/                   # SQL скрипты
│   ├── create.sql         # Создание схемы БД с индексами
│   ├── insert_queries.sql # Примеры INSERT запросов
│   ├── update_queries.sql # Примеры UPDATE запросов
│   ├── delete_queries.sql # Примеры DELETE запросов
│   ├── select_queries.sql # Примеры SELECT запросов
│   ├── test_data.sql      # Тестовые данные
│   └── indexes_report.md  # Отчет об индексах

```

## Реализованные функции

### 1. Авторизация пользователей
- Регистрация новых пользователей
- Вход по логину и паролю
- JWT токены для аутентификации
- Контроль активного состояния пользователей

### 2. Управление пользователями
- Создание, редактирование и удаление пользователей
- Разграничение ролей (пользователь/администратор)
- Просмотр профилей пользователей
- Поиск пользователей

### 3. Публикация контента
- Создание постов
- Создание ответов на посты
- Добавление лайков
- Добавление в избранное
- Просмотр ленты новостей

### 4. Социальные взаимодействия
- Подписка на пользователей
- Просмотр подписок и подписчиков
- Личная лента на основе подписок

### 5. Личные сообщения
- Создание приватных чатов между двумя пользователями
- Отправка и получение сообщений
- Прикрепление файлов к сообщениям
- Статус прочтения сообщений

### 6. Модерация и жалобы
- Создание жалоб на пользователей и посты
- Рассмотрение жалоб администраторами
- Выдача и снятие банов
- Журналирование действий пользователей

## Структура базы данных

### Основные сущности

1. **user** - Пользователи системы
   - id, nick, password_hash, avatar_id, is_admin, created_at

2. **post** - Посты и комментарии
   - id, text, author_id, answer_to_id, created_at

3. **subscription** - Подписки между пользователями
   - user_from_id, user_to_id, created_at

4. **like** - Лайки постов
   - post_id, user_id, created_at

5. **favorite** - Избранные посты
   - user_id, post_id, created_at

6. **chat** - Чаты между пользователями
   - id, first_user_id, second_user_id, created_at

7. **message** - Сообщения в чатах
   - id, author_id, chat_id, content, file_path, created_at

8. **media** - Медиа-файлы
   - id, file_path

9. **report** - Жалобы на пользователей и посты
   - id, author_id, target_user_id, post_id, comment, is_reviewed, created_at

10. **ban** - Баны пользователей
    - id, banned_user_id, admin_id, report_id, start_date, end_date, reason

11. **log** - Журнал действий пользователей
    - id, time, user_id, details

## API документация

### Аутентификация
```
POST /auth/register - Регистрация пользователя
POST /auth/login - Вход в систему
```

### Пользователи
```
GET /users/profile - Получить свой профиль
GET /users/{userId} - Получить профиль пользователя
PUT /users/profile - Обновить профиль
GET /users/search - Поиск пользователей
GET /users/{userId}/followers - Получить подписчиков
GET /users/{userId}/following - Получить подписки
```

### Посты
```
POST /posts - Создать пост
GET /posts/{postId} - Получить пост
GET /posts/feed - Получить ленту новостей
GET /posts/user/{userId} - Посты пользователя
PUT /posts/{postId} - Редактировать пост
DELETE /posts/{postId} - Удалить пост
POST /posts/{postId}/like - Поставить лайк
DELETE /posts/{postId}/like - Убрать лайк
POST /posts/{postId}/favorite - Добавить в избранное
DELETE /posts/{postId}/favorite - Убрать из избранного
GET /posts/favorites - Избранные посты
```

### Подписки
```
POST /subscriptions/{targetUserId} - Подписаться
DELETE /subscriptions/{targetUserId} - Отписаться
GET /subscriptions/{targetUserId}/status - Проверить подписку
GET /subscriptions/counts - Количество подписок/подписчиков
```

### Чаты
```
GET /chats - Получить список чатов
POST /chats/with/{targetUserId} - Создать чат
GET /chats/{chatId}/messages - Получить сообщения
POST /chats/{chatId}/messages - Отправить сообщение
PUT /chats/{chatId}/messages/{messageId} - Редактировать сообщение
DELETE /chats/{chatId}/messages/{messageId} - Удалить сообщение
```

### Жалобы и модерация
```
POST /reports/user/{targetUserId} - Пожаловаться на пользователя
POST /reports/post/{postId} - Пожаловаться на пост
GET /reports/{reportId} - Получить жалобу
GET /admin/reports - Получить все жалобы (админ)
PUT /admin/reports/{reportId}/reviewed - Отметить рассмотренной
POST /admin/reports/{reportId}/ban - Выдать бан
DELETE /admin/bans/{banId} - Снять бан
GET /admin/bans - Получить активные баны
```

## Настройка и запуск

### Требования
- .NET 9.0 SDK
- PostgreSQL 12+
- Docker (опционально, для контейнеризации)

### Конфигурация
В файле `appsettings.json` настроить подключение к БД:
```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Host=localhost;Port=5432;Database=socnet;Username=admin;Password=123456;"
  },
  "Jwt": {
    "Key": "your_secret_key_here",
    "ExpiryMinutes": 60
  }
}
```

### Запуск
1. Создать базу данных PostgreSQL
2. Выполнить скрипт `SQL/create.sql` для создания схемы
3. Выполнить скрипт `SQL/test_data.sql` для загрузки тестовых данных
4. Запустить приложение: `dotnet run`
5. Открыть Swagger UI по адресу `https://localhost:5001/swagger` для тестирования API

### Тестирование API через Swagger
Проект включает Swagger UI для удобного тестирования всех API эндпоинтов:

1. После запуска приложения откройте `https://localhost:5001/swagger`
2. Сначала зарегистрируйтесь или войдите в систему через эндпоинты `/auth/register` или `/auth/login`
3. Скопируйте JWT токен из ответа на вход
4. Нажмите кнопку "Authorize" в Swagger UI и введите токен в формате `Bearer YOUR_TOKEN_HERE`
5. Теперь вы можете тестировать все защищенные эндпоинты

Тестовые учетные данные:
- Логин: `alice`, Пароль: `password123`
- Логин: `admin`, Пароль: `password123` (администратор)

### Docker
Проект включает docker-compose.yml для запуска с PostgreSQL:
```bash
docker-compose up -d
```

- JWT токены с ограниченным временем жизни
- Проверка ролей для административных функций

### Авторизация

- Предотвращение SQL-инъекций через параметризованные запросы
